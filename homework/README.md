## Задание 1 — Глубина сети

### 1.1 Сравнение глубины
| Архитектура | Ожидаемое поведение |
|---|---|
| 1 layer (linear) | Низкая точность, нет нелинейности |
| 2 layers | Существенный скачок качества |
| 3 layers | Оптимальный баланс |
| 5 layers | Начало переобучения |
| 7 layers | Переобучение или деградация |

### 1.2 Регуляризация на глубокой сети
Dropout и BatchNorm уменьшают разрыв между train/test accuracy. Оптимальна комбинация Dropout(0.3) + BN.

## Задание 2 — Ширина сети

### 2.1 Сравнение ширины (3 слоя)
- Узкие слои недостаточно выразительны, широкие — избыточны
- Средняя ширина [256,128,64] обычно оптимальна для данного датасета

### 2.2 Grid Search
Heatmap `глубина × ширина` показывает, что нет смысла бесконечно увеличивать ни глубину, ни ширину.

## Задание 3 — Регуляризация

### 3.1 Сравнение техник
- **Без регуляризации**: максимальный train, высокий разрыв
- **Dropout 0.3**: хорошее обобщение
- **BatchNorm**: стабильное обучение
- **Dropout + BN**: лучший баланс
- **L2**: мягкая регуляризация, полезна при больших весах

### 3.2 Адаптивная регуляризация
- Линейное уменьшение p Dropout: высокая регуляризация в начале, меньше — в конце
- BN с малым momentum (0.01): медленная адаптация статистик, устойчивее на малых батчах
